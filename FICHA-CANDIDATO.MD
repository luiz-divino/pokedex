Seção 1: Instruções para rodar

Como rodar o projeto?
 - Inicie o servidor de desenvolvimento passando o comando:
npm run dev

Quais variáveis de ambiente são necessárias?
Nenhuma variavel de ambiente foi necesaria visto que consumi uma API publica.

Como instalar dependências?
 - No terminal, instale as dependencias passando o comando:
 npm install 

Seção 2: Decisões de design

Por que você escolheu essa estrutura de pastas?
 Eu escolhi essa estrutura pois é uma arquitetura padrão para projetos em react e que me adaptei ao longo das minhas praticas, além disso, por ser padrão é algo que me ajude a trabalhar com outros desenvolvedores e no entendimento do codigo.

Qual foi a maior dificuldade que você encontrou e como superou?
 A tarefa de ter que implementar imagens visto que as mesmas não vem na API. Eu já tinha tido contato com essa api e com o repositorio do pokedex, isso me ajudou pois os IDS são universais para todas as URLs do repositorio, ainda sim, precisei pesquisar e usar ferramentas para cumprimentar a minha ideia, pois tive dificuldade para tirar os espações da URLs e consumir apenas o ID.

O que você não teve tempo de fazer (dentro do timebox) e como você faria se tivesse mais tempo?
Os filtros para consultar o pokemon pelo nome e adiciona-los como favoritos, acabei perdendo algum tempo no processo e apesar de tentar eu não consegui encontrar uma solução para esse problema, acabei pesquisando meios de fazer isso mas nada que funcionasse concretamente, além disso, eu nunca tinha salvo informações em cache antes e pensei em usar o localStorage mas o tempo estava curto.
